# Plans

Managed by `memex-md`. Each plan is a design artifact written before implementation. Plans travel with the repo so the team shares context.

## Plans

- [2026-05-13-command-first-investigation.md](2026-05-13-command-first-investigation.md) — Command-first investigation. Shift the agent's default context-gathering from `read_file` to `run_command` (mirroring Claude Code). Five stages: system-prompt directive rewrite, safe-command allowlist that auto-approves `git status` / `ls` / `grep` / `find`, reasoning-brief `investigationPlan` field, divergence heuristic `no_investigation_before_write`, free-tier investigation budget + knowledge entries. Composes on top of model-lock-and-portable-reasoning. *(draft)*
- [2026-05-07-model-lock-and-portable-reasoning.md](2026-05-07-model-lock-and-portable-reasoning.md) — Model lock-in + portable reasoner + brief enforcement. Trim `MODEL_FALLBACK_CHAINS` so explicit picks (claude-*, gemini-*) don't silently swap providers; wire reasoning briefs into Anthropic + OpenRouter system prompts (today only Gemini reads them); make the reasoner backend pluggable (Gemini Flash / Anthropic Haiku / OpenRouter free) so non-Gemini main models stay reasoning-capable when Gemini is down; confidence-aware directive framing on briefs. *(in-progress)*
- [2026-05-07-phase-17-adaptive-prompts-and-preferences.md](2026-05-07-phase-17-adaptive-prompts-and-preferences.md) — Phase 17: inferred user-preference engine. Pure helpers infer language / style / output-format / decision-default preferences from session history; `recordPreference()` writes; LEARNED_MEMORY prompt section grows a preferences sub-block; language + style become system-prompt directives at high confidence. *(draft)*
- [2026-05-07-phase-18-pre-task-deep-reasoning-and-complexity.md](2026-05-07-phase-18-pre-task-deep-reasoning-and-complexity.md) — Phase 18: pre-task deep reasoning + complexity-gated taskPlan. Move `analyzeTaskComplexity` to pre-flight; add `pre_task_reasoning` + `task_confirmation` Flash pipelines that produce `whyThisApproach` + alignment check before taskPlan emission; gate taskPlan emission to `implement` + complexity ≥ medium. *(draft)*
- [2026-05-07-phase-19-task-display-selectivity.md](2026-05-07-phase-19-task-display-selectivity.md) — Phase 19: Claude-style task-box visibility. Server emits `intent_classified` early; cli reducer holds `runIntent`; AgentStream gates the task-header render on `intent === "implement"`; Header gets a muted "thinking through it" indicator for non-implement runs with internal task structure. *(draft)*
- [2026-05-05-phase-9-ink-ui-rewrite.md](2026-05-05-phase-9-ink-ui-rewrite.md) — Phase 9: replace cli-client's `console.log` + `readline` rendering with an Ink (React-for-terminal) component tree. Live in-place task list, multiline chat input with history, permission prompt as a modal with embedded structured diff, animated spinner with cycling verbs. Logic untouched — purely the rendering layer. *(superseded by Phase 12)*
- [2026-05-02-phase-8-persistent-reasoning-memory.md](2026-05-02-phase-8-persistent-reasoning-memory.md) — Phase 8: persistent reasoning memory with active anti-staleness. New `.sysflow-memory.md` (sibling of user-written `.sysflow.md`) auto-records decisions/summaries/corrections; read-time validators (file-existence, dep-existence, age) gate every entry; auto-confirmation/contradiction; `/memory` and `/remember` slash commands; compaction at 100 KB.  *(in-progress)*
